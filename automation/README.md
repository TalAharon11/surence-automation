# אוטומציית עדכון דוחות יומיים

סקריפט Playwright שקורא דוח יומי (Excel) ומעדכן את המשימות באתר הדמה לפי כללי האוטומציה.

## מה הסקריפט עושה

1. קורא קובץ Excel עם המבנה הסטנדרטי (6 עמודות):
   `שם | תעודת זהות | מספר קופה | האם הטפסים התקבלו תקין | הערות | צפי ניוד`
2. נכנס לאתר הדמה ב-Chromium (Playwright)
3. עבור כל שורה בדוח, מזהה את המשימה לפי **תעודת זהות + מספר קופה** ומעדכן לפי 3 המקרים:
   - **מקרה 1 — צפי ניוד**: תקין=כן + יש תאריך צפי → סטטוס "בטיפול", תאריך טיפול = תאריך הצפי, הערה "צפי ניוד בתאריך: DD/MM/YYYY"
   - **מקרה 2 — ממתין להפקדות**: תקין=כן + הערה "ממתין להפקדות" → סטטוס "בטיפול", תאריך טיפול = ה-15 בחודש העוקב, הערה "ממתין להפקדות"
   - **מקרה 3 — לא תקין**: תקין=לא → סטטוס "נדחה", תאריך טיפול = היום, הערה לפי שדה "הערות"
4. מפיק קובץ `audit-report.txt` קריא בעברית עם סיכום מלא.

## הפעלה

```bash
cd automation
npm install
npm run install-browsers   # פעם ראשונה בלבד

# הרצה מול אתר הדמה (URL ברירת מחדל)
node automate.js reports/harel_daily_report.xlsx

# הרצה מול URL אחר
SITE_URL=https://my-site.lovable.app node automate.js reports/menora_daily_report.xlsx

# הרצה במצב headless (לרקע / Docker / n8n)
HEADLESS=true node automate.js reports/harel_daily_report.xlsx
```

## פלט

- `audit-report.txt` — דוח קריא בעברית עם סיכום וכל המשימות שטופלו / נדחו / נכשלו.
- `audit-report.json` — אותו מידע במבנה מובנה לשימוש על ידי n8n / מערכות אחרות.

## מבנה לעבודה עם n8n

- **שלב פיתוח (עכשיו)**: Manual Trigger → Execute Command → קורא קובץ מקומי.
- **שלב Production**: Gmail Trigger → שמירת הקובץ המצורף → Execute Command → הסקריפט הזה → שליחת ה-Audit Report במייל לסוכן.

הסקריפט לא תלוי במקור הקובץ — הוא רק צריך נתיב לקובץ Excel תקין.
